Reference

Public MCP tool reference

This page documents the stable public tool surface exposed by the Minestom MCP Server. It is designed to be readable by humans in a browser and easy to map back to the repository’s markdown and AI index files.

Reference Notes
  • Tool names are shown exactly as exposed over MCP.
  • Inputs and outputs are intentionally summarized rather than exhaustively schema-dumped.
  • Build and environment tools are repo-aware and JVM-aware.

ping

Health check for MCP reachability.

Meta

Inputs

  • message optional string to echo back

Returns

  • ok
  • timestamp
  • echoedMessage

get_server_info

Server metadata, tool inventory, and knowledge-catalog coverage.

Meta

Inputs

  • includeDependencies optional boolean

Returns

  • name, version, description
  • toolCount and availableTools
  • runtime
  • knowledgeCatalog
  • optional dependencies

inspect_minestom_environment

Workspace-level Minestom and JVM environment inspection.

Environment

Scans the current workspace or a provided repoRoot, including subdirectories. This is the right tool when you need to figure out whether a tree actually looks like a Minestom workspace and where the interesting modules live.

Key Inputs

  • repoRoot optional path to inspect

Key Returns

  • buildTool, buildFiles, projectRoots
  • languages, sourceRoots, packageNamespaces
  • entrypointFiles, detectedTopics, detectedApiSymbols
  • jvmProject, runDirectory, runDirectories
  • existingLibraries, notes
Important: run/ is treated as a strong development signal, not as the only possible server location.

inspect_minestom_build

Gradle and Maven specific build interpretation for Minestom workspaces.

Build

Use this when you need build-system detail instead of high-level environment detection. It understands common Gradle and Maven structures and returns module-level details.

Gradle Coverage

  • plugins { ... }
  • dependencies { ... }
  • gradle.properties
  • gradle/libs.versions.toml
  • wrapper scripts like gradlew
  • task or script names

Maven Coverage

  • pom.xml dependencies
  • pom.xml plugins
  • pom.xml properties
  • wrapper scripts like mvnw
  • command suggestions such as package and dependency:tree

Module Returns

  • moduleRoot, buildTool, buildFiles, wrapperFiles
  • plugins, dependencies, variables, commands
  • notes and optional versionCatalog

explain_minestom_pattern

Curated pattern explanation grounded in official Minestom guidance.

Knowledge

Inputs

  • topic from bootstrap, instances, events, commands, scheduler, threading
  • goal optional context string

Returns

  • pattern explanation
  • key classes
  • lifecycle notes
  • common pitfalls
  • official links

lookup_minestom_api

Curated Minestom API lookup for common server patterns.

Knowledge

Returns the most relevant curated matches for a symbol or topic and links each result back to the Javadocs.

plan_minestom_feature

Implementation planning for Minestom features and JVM languages.

Planning

Produces file outlines, key APIs, implementation steps, thread-safety notes, and verification steps for common feature types such as commands, event listeners, instance setup, and scheduled tasks.

review_minestom_design

Design review against Minestom’s manager, event, instance, and threading patterns.

Planning

Useful before implementation when you want a high-signal review of fit, gaps, risky assumptions, and thread or tick concerns.

suggest_minestom_libraries

Curated ecosystem recommendation tool with optional live GitHub results.

Ecosystem

This tool always starts with curated Minestom ecosystem knowledge. When repoRoot is provided, it first inspects the workspace and uses detected Minestom and JVM signals to rank suggestions and suppress libraries that already appear to be present.
