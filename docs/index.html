Documentation built for both humans and AI clients.

This server exposes a curated Minestom MCP surface for pattern explanation, API lookup, repository and environment scanning, Gradle and Maven build inspection, design review, implementation planning, and ecosystem discovery.

  • Publish-ready Pages deployment via GitHub Actions.

Repository-aware by default

The environment tooling treats the inspected directory as a workspace, scans subdirectories, detects candidate project roots, and treats run/ as a strong development signal instead of a hardcoded server location.

JVM-native inspection

Because Minestom lives in the JVM ecosystem, the docs and tooling explicitly account for Gradle, Maven, wrapper scripts, Java/Kotlin layouts, version catalogs, dependencies, plugins, and project properties.

AI-friendly access

The docs include a concise landing page for humans, a detailed tool reference, and llms.txt so AI systems can discover the right reference documents quickly.
